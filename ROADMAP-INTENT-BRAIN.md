## Executive summary

NexoraAI is already the category leader on its own axes — local-first generation, debug/repro engine, agent layer (trust, diff-review, comment-to-steer, knowledge base), 157-provider BYO-key, local vision-in + image-gen, Universal Agent with intent directives. The 10-agent competitor sweep (Cursor, Windsurf/Devin, Copilot, Cline/Roo/Aider, Bolt/v0/Lovable/Replit, Antigravity/Kiro/Spec-Kit, plus the local-runtime and context-engineering literature) shows the real gaps are NOT more surface area but three structural levers: (1) it never asks whether it understood the intent before spending minutes on a build, and it can't retrieve the RIGHT context on real repos (it dumps whole files with a silent 400-file cliff); (2) it under-exploits the very engine it already runs — the spawned llama-server supports speculative decoding, KV-slot cache, embeddings, and lazy/triggered grammars, all confirmed unwired in source, leaving free 1.5-2.5x speed, ~93% faster resume, real retrieval, and near-100% directive-JSON reliability on the table; (3) its intent directives ([RUN]/[IMG]/[BUILD]…) are emitted as free-form text a 3B mangles, and the sd-server it already spawns exposes img2img/inpaint/edit that localImageService never calls. The biggest opportunities, in order: an offline retrieval brain (repo-map + semantic index + [SEARCH]/[SYMBOL]), a calibrated intent gate that asks-or-forks before building, engine-native turbo/resume/grammar upgrades, a persistent editable spec artifact, and cheap multimodal reach (OCR via the already-running VL sidecar, image editing via the already-running sd-server, voice dictation). These sharpen "do everything from intent" while staying 100% on-device.

## Top-5 highest-leverage moves

1. Offline retrieval brain — tree-sitter repo-map + PageRank + GGUF code-embedding index (via the already-spawned llama-server --embedding) + [SEARCH]/[SYMBOL] directives, replacing the silent 400-file whole-file dump. Biggest correctness+cost win and the thing that makes NexoraAI usable on real existing repos, not just greenfield.
2. Triggered-grammar directive & tool-arg fidelity — lazy GBNF (already have editGrammar.ts, zero trigger-grammar today) forcing [RUN]/[IMG]/[MCP]/tool payloads to valid schema the instant the model decides to act. Makes the Universal Agent's directives near-100% parseable on a 3B; small effort, foundational for every other intent feature.
3. Intent Gate + Interpretation Fork — a calibrated ask-or-show-options pass before any byte is written (UA-Multi 54.8%→69.4%, proven on 4B). Kills the silent-guess-then-build-then-repair cycle that burned the Zeytin/Volta live tests; the deepest expression of 'operate from understood intent.'
4. Engine turbo & instant resume — auto-paired speculative decoding (1.4-2.5x free) + KV-slot cache (~93% faster session/project resume), both unwired despite the runtime supporting them. Directly fixes slow multi-file builds and watchdog hangs while reusing the VRAM advisor + multi-model download the app already has.
5. Multimodal reach on running sidecars — OCR via the live Qwen-VL sidecar and image editing (img2img/inpaint/[EDIT]) via the sd-server that already exposes /edits but is never called, plus Z-Image Turbo flagship. Near-zero new engine, immediately widens what intents the model can act on.

## Themes (deduped, ranked)

### Offline Code Intelligence & Retrieval (the repo brain)
*Impact: **high** · Effort: **L** · Local-first fit: **native** · Seen in: Cursor (@codebase, Instant Grep + semantic), Aider (tree-sitter + PageRank repo map), Roo Code / Kilo (Ollama+Qdrant semantic index), Continue.dev (local LanceDB+MiniLM), Serena (LSP symbol tools)*

Confirmed in source: the app sends WHOLE files with a hard 400-file / 200KB cap and silently drops the rest — no embeddings, no tree-sitter, no repo map, no symbol nav (grep returns 0 files). This is the single largest correctness+cost lever for real (non-greenfield) repos. Give the model a fully-local three-layer retrieval stack — lexical (ripgrep), structural (tree-sitter def/ref graph + PageRank, signatures-not-bodies), semantic (GGUF code-embedder served by the SAME llama-server via --embedding) — plus symbol-level surgical edits, exposed as model-emitted [SEARCH]/[SYMBOL] directives and user @-file/@-symbol pins. Replaces 'dump 400 files' with 'retrieve the ~30k tokens that matter.'

**Intent angle:** The model chooses the query and expresses layer preference in plain language exactly like the existing [RUN]/[IMG] demote-heuristics-to-hints pattern; PageRank is personalized by the user's current message so retrieval tracks intent turn-by-turn instead of static file lists.

**Concrete features:**
- Tree-sitter def/ref symbol graph + personalized PageRank ranked skeleton (10x message-mentioned, 50x in-chat files), reparse-on-save incremental
- Offline GGUF code-embedding index (nomic-embed-code / jina-code-1.5b) via llama-server --embedding, on-disk vectors, Merkle-hash incremental re-index, cAST function-boundary chunking
- Model-emitted [SEARCH] directive fanning out to ripgrep + ast-grep + semantic, merged into a token-budgeted block; layer preference stated in natural language (hint, not regex)
- [SYMBOL] surgical edits: find_symbol / find_referencing_symbols / go_to_definition / replace_symbol_body (tree-sitter native, optional LSP upgrade), gated by existing per-hunk diff review
- @-file/@-symbol/@-folder pinning with a context-chips bar + token cost; replace the 400-file cliff with ranked selection + 'what got dropped and why'

### Intent Fidelity: ask-or-fork before building
*Impact: **high** · Effort: **M** · Local-first fit: **native** · Seen in: Cursor Plan Mode (clarifying questions), 'Ask or Assume?' UA-Multi, multi-interpretation single-response (arxiv 2511.10453)*

NexoraAI's flow is plan→build→verify with no metacognitive gate — the Zeytin/Volta live tests show it executing 'make it modern' / 'add auth' on silent assumptions and only discovering the mismatch after minutes of build+repair. Add a decoupled, calibrated intent pass (UA-Multi: 54.8%→69.4% resolve) that, only when a request is underspecified, EITHER asks the smallest necessary clarifying question OR emits up to 5 self-contained interpretation cards (proven on Qwen3-4B — squarely on-device) for a one-click pick. Stays silent on clear tasks.

**Intent angle:** The purest form of intent-based operation: the model DECIDES whether it actually understands yet and only acts when it does — metacognition about intent, driven by understanding, never keyword patterns.

**Concrete features:**
- Intent Gate: cheap second local pass reads goal / what's missing / what must be clarified; outputs proceed-or-ask; calibrated to ~2-3 questions per genuinely vague task, near-zero on clear ones
- Interpretation Fork: up to 5 interpretation-answer cards in one structured response, each with a one-line build preview; collapses to a single card when unambiguous
- Wire the gate's decision into the existing plan-mode UX so ambiguity is resolved before any byte is written

### Engine-Native Turbo, Resume & Grammar (exploit the llama-server we already spawn)
*Impact: **high** · Effort: **M** · Local-first fit: **native** · Seen in: LM Studio (draft-model UI, speculative decoding), llama.cpp (--slot-save-path, lazy grammars, --jinja tools), Jan (MTP multi-token prediction), XGrammar (near-free constrained decoding)*

Source confirms none of these are wired despite node-llama-cpp 3.19 + llama-server supporting them: no speculative decoding (0 draft-model refs), no KV-slot cache (0 slot-save-path), no --embedding, and GBNF grammar exists ONLY for edit-format (editGrammar.ts) — never lazy/triggered on directives/tool-args (0 trigger-grammar refs). Free wins: 1.4-2.5x generation speed, ~93% faster session/project resume, and near-100% parseable directives on a 3B. Directly attacks the documented watchdog hangs and slow multi-file builds.

**Intent angle:** Grammar constrains the FORM of an already-intended action without keyword-deciding the action — the model keeps final say (routing stays a hint); faster/warm-KV loops keep the understand→do→verify cycle interactive instead of timing out.

**Concrete features:**
- Auto-paired speculative decoding: load a same-family draft GGUF (Qwen 0.5B→7/14B, Llama-3.2-1B→3.1-8B) with --model-draft/--draft-max; 'Turbo' toggle + live accept-rate; VRAM advisor accounts for draft footprint
- Instant resume via --slot-save-path + default host-RAM prompt cache: map each session/project to a stable slot id, save KV on idle, restore on reopen (session + proje-gecmisi.md + knowledge is the biggest reusable prefix)
- Triggered-grammar directive & tool-arg fidelity: model reasons freely, then the instant it emits [RUN]/[IMG]/[MCP]/tool-call the payload is constrained-decoded to a valid schema; one hidden repair turn only on SEMANTIC failure
- Phase-aware reasoning-token budget (quick fix vs deep plan) + asymmetric KV-quant profile (K low/V high) chosen by the advisor, with a guard against the tool-calling degradation aggressive KV-quant causes
- Tool-template health check on model load: probe chat_template_tool_use, badge native-vs-generic, measure valid-call rate to inform escalation

### Durable, Editable Spec & Governance Artifacts
*Impact: **high** · Effort: **L** · Local-first fit: **native** · Seen in: Amazon Kiro (requirements/design/tasks + EARS + steering), GitHub Spec Kit (/constitution /specify /plan /tasks), Windsurf (bidirectional plan.md), Antigravity (brain/ decisions), Copilot (AGENTS.md interop)*

The live plan is ephemeral (artifactDocsService/walkthrough) and consumed, not a persisted file the user edits before code — the plan-dropped-Galeri and restaurant-hallucination misses are exactly what a 15-second editable-plan read catches. Promote it to living markdown: requirements.md (EARS acceptance criteria) / design.md / tasks.md that both human and model mutate, an approval-gated project constitution.md the verifier enforces, an ADR-lite decision log read before planning, and per-criterion traceability so 'verified' finally measures the wanted requirement (fixes the noted 'verified without the line' bug). Also consume the cross-tool AGENTS.md/CLAUDE.md standard.

**Intent angle:** The spec IS captured intent in the user's own words and editable by them; the model reconciles code to the spec each turn instead of re-guessing a one-line prompt — durable, versioned, human-and-machine-readable intent.

**Concrete features:**
- Living spec files under .nexora/specs/<feature>/ (requirements/design/tasks) with EARS 'WHEN…SHALL…' criteria, approval gates + a Quick-Plan escape, re-read each turn — spec drives generation, not chat scrollback
- Acceptance-criteria traceability: stable ids, tasks cite ids, self-review/repro maps checks back to ids, live met/unmet/unverified scorecard; walkthrough generated FROM the scorecard
- Project constitution.md (strict TS, Tailwind-not-Bootstrap, a11y baseline) — approval-gated, always feeds planning, violations are blocking findings not style hints (global→project precedence like KURALLAR)
- Architectural-decision log (ADR-lite): 'chose Zustand over Redux, because' entries written on forks, always-include-at-plan-time, explicit supersede
- AGENTS.md / CLAUDE.md interop: read root + nested files, fold into the hierarchical rules stack with a source chip in the Knowledge panel

### Scoped Context Injection & Auto-Learned Memory
*Impact: **high** · Effort: **M** · Local-first fit: **native** · Seen in: Windsurf (4-mode rules, auto Memories + smart retrieval), Kiro (steering: always/conditional/manual/auto), Cline (.clinerules vs /workflows token accounting), Cursor (Memories, .mdc scoped rules)*

Rules/knowledge are effectively always-on and budgeted into every turn, and the first item is silently trimmed under pressure (a documented bug). The model also can't volunteer a general 'remember this' fact — learning is confined to fixed repair-pattern throat-points. Add per-entry inclusion modes (always / glob-match / manual #ref / AUTO model-decides-relevance) so the KB scales past its 30-item cap without prompt bloat, a [REMEMBER]-driven memory for arbitrary discovered project truths ('uses pnpm', 'API base=/v2') with local retrieval, and a behavioral-rules vs procedural-workflow token-cost distinction. Note: [REMEMBER] directive already exists in source (7 refs) — this deepens it from fixed-throat to model-volunteered + retrieval-surfaced.

**Intent angle:** AUTO mode is intent-matching by construction — the model judges which knowledge applies to THIS request rather than a fixed always-inject list; [REMEMBER] is the model deciding what's worth keeping from understanding the conversation, same philosophy as [RUN]/[IMG].

**Concrete features:**
- Four inclusion modes on each KB/KURALLAR entry via front-matter: Always, Glob file-match, Manual #ref, AUTO (tiny local relevance judgment before spending budget)
- Model-volunteered memories: extend the existing [REMEMBER] emit to any conversational point, workspace-scoped, type chip + hit counter, surfaced via on-device embed/keyword retrieval (no @mention)
- Retrieval over the whole KB (repair patterns + rules + proje-gecmisi.md) so only top-k relevant chunks enter the budgeted block instead of dump-and-truncate
- Surface per-turn token cost of persistent rules; separate always-on behavioral rules from on-demand procedural workflows

### Live Preview as Behavioral Proof & Point-and-Steer
*Impact: **high** · Effort: **L** · Local-first fit: **native** · Seen in: Replit Agent 3 (REPL behavior self-test), v0 / Lovable (click-to-select + Design Mode), Windsurf (send element / send error), Antigravity (browser-in-the-loop), Bolt (autonomous error self-fix)*

Visual self-review is agent-initiated screenshot review — it proves 'renders', not 'works', shipping Potemkin UIs (dead onClick, mocked data, non-persisting state). The CDP path already exists (debugInspect.ts). Upgrade it: a REPL/CDP-driven behavior prover that clicks, submits, asserts handlers fired and state survives reload, feeding failures into the existing repro→auto-fix loop; plus click-an-element-in-the-preview to make it the exact edit target (reusing comment-to-steer + source anchors), and a deterministic Design-Mode panel that writes Tailwind classes back WITHOUT a model turn (reusing the class-slot enforcement machinery).

**Intent angle:** The test plan and edits are derived from what the app is SUPPOSED to do (contract/brief) and from the user pointing at 'THIS element' — removing 'which element' ambiguity so the model spends reasoning on the change, not on locating the target; grounds intent in real DOM/error facts like the Universal Agent's real-machine-facts approach.

**Concrete features:**
- Behavior Prover subagent: injects an automation shim over the live preview, reads augmented DOM/ARIA + console/server logs (+ local DB when present), derives test intent from the contract, action-observe loops until confirmed or emits a concrete 'onClick missing / data mocked / nothing persisted' finding
- Point-and-Steer: 'select' toggle → click a DOM node → capture stable locator + source file:line/JSX span → pin as '← EDIT TARGET' on the next message
- Send-element / send-error handoff: click a rendered node or a captured console error and hand THAT precise target to the next turn
- Design Mode: v0-style typography/spacing/color/radius/shadow panel + Layers tree; trivial tweaks apply deterministically to Tailwind classes (instant, free), only structural asks route to the model — every edit diffable/revertable

### Self-Healing Verify Loop & Post-Edit Contract
*Impact: **high** · Effort: **M** · Local-first fit: **native** · Seen in: Aider (lint+test self-heal, per-model edit format), Bolt (Diffs — change only what changed), Cursor (afterFileEdit hooks), Copilot agent (reads test output, self-heals)*

NexoraAI repro-verifies with its OWN internal checks and lazy-pretties implicitly, but never runs the PROJECT's own linter/typechecker/test-cmd scoped to the edited files after every write, and there's no per-model edit-format policy with graceful fallback (a 3B silently corrupting a file vs quietly succeeding). Add an explicit afterEdit contract derived from package.json/pyproject, an adaptive whole-file-for-weak / search-replace-for-strong format policy that downgrades on apply-failure and learns per model, and a hard diff-only iteration contract so small changes can't balloon into whole-file rewrites (protecting the hard-won fidelity).

**Intent angle:** The pass/fail signal comes from the project's REAL tooling and the model reads the failure and decides the fix — evidence-driven self-correction consistent with the existing deterministic-learning-on-evidence design; the harness picks the safest wire format per model, keeping the intent layer free of format-wrangling.

**Concrete features:**
- afterEdit contract: auto-detect format+lint+typecheck+test from the repo, run scoped to edited paths after every write, feed non-zero output into the existing lastBuildError→one-word-fix loop, evidence-logged in the Motor panel
- Per-model adaptive edit format with automatic downgrade to whole-file on anchor-miss/malformed-block, recorded in the KB so the same model reuses the working format
- Diff-only iteration contract: iterations must return anchored SEARCH/REPLACE hunks against real bytes (reuse the Dice-similarity anchor engine); UI flags when a small ask becomes a full-file rewrite

### Multimodal Reach on Already-Running Sidecars (OCR, image edit, voice)
*Impact: **high** · Effort: **M** · Local-first fit: **native** · Seen in: Ollama desktop (image input), ComfyUI/Krita AI (local image edit), (none ship integrated dictation+TTS+image-gen — genuinely differentiated)*

Highest impact-per-effort: three inputs/outputs the app can add with near-zero new engine. The Qwen2.5-VL sidecar already runs (visionService.ts) → OCR/document intake is a prompt away. The sd-server already spawns and exposes init_image/mask_image/control_image/edits, but localImageService only POSTs /generations (confirmed) → wire img2img/inpaint/instruction-edit + add Z-Image Turbo (4GB-capable, frontier quality; already 3 catalog refs) as the flagship. whisper.cpp + Kokoro-82M add on-device dictation and read-aloud via the same sidecar pattern — a full local voice loop no competitor ships.

**Intent angle:** Each adds a NEW modality to the same intent path — speak a task or drop an image and the model reads the content and decides what to do ([RUN]/[BUILD]/[EDIT]); generate-vs-edit is a model decision like generate-vs-not, never a filename keyword.

**Concrete features:**
- OCR / document intake: drag-drop a screenshot/PDF-page/photo → send to the running VL sidecar → text or structured Markdown into the prompt or a project file (optional Tesseract/PaddleOCR CPU fallback)
- Local image EDITING: extend localImageService to POST init_image/strength (img2img), mask_image (inpaint), multipart /v1/images/edits; add Qwen-Image-Edit-2511 GGUF; new [EDIT] directive sibling to [IMG]; right-click any asset → 'Edit with prompt' / brush a mask
- Z-Image Turbo as flagship local model (8-9 steps/CFG 1.0, 4GB-capable) with corrected recognizer + fits-your-GPU badge; optional ControlNet/LoRA advanced panel
- Voice dictation via whisper-server sidecar (VAD, OpenAI-compat /v1/audio/transcriptions, multilingual to match the 10-language UI); push-to-talk in the prompt box
- Read-aloud via Kokoro-82M ONNX (plans/results/walkthroughs); pairs with dictation for a full hands-free local voice loop

### Long-Horizon Orchestration: folding, subagents, live steering & rewind
*Impact: **medium** · Effort: **L** · Local-first fit: **native** · Seen in: Roo/Kilo (Boomerang isolated subtasks), Copilot (parallel subagents), Claude Code (h2A steering queue), AGDebugger/LangGraph (time-travel), Antigravity/Zed (multi-worktree Agent Manager)*

Long builds still accrete history in one window (the Hero=Navbar clone that needed isolate flags), Stop doesn't truly cancel the busy llama-server job, queued turns race the repair chain, and the Motor timeline is descriptive not replayable. Add hierarchical context-folding (each subtask an isolated sub-context, folded back as a decision-only digest), parallel-friendly context-isolated subagents (read-only explorer/reviewer returning conclusions only), a real h2A steering channel that folds a correction into the RUNNING turn with a cancel that reaches the server, a consecutive-action budget circuit-breaker, and AGDebugger-style trajectory rewind reusing existing checkpoints + the logRepair choke point. Parallel multi-agent worktrees are the heavier, lower-priority slice.

**Intent angle:** Folding preserves each subtask's intent (contract/decisions) while discarding noise so downstream turns reason over goals not scrollback; the steering channel keeps the loop intent-responsive in real time; rewind exposes and lets the user CORRECT the agent's inferred intent at the exact point it diverged.

**Concrete features:**
- Context-folding orchestrator: each section/subtask in an isolated context (extend options.isolate), folded into a decision-only summary; a 'research fold' returns a 1-2k brief
- Context-isolated subagents (read-only explorer / reviewer / test-reader, write-barred via trust, infer:false on destructive ops); parallel with API, queued-but-isolated locally
- Live steering channel: 'Steer now' vs 'Queue' vs 'Stop', corrections fold at the next safe step, and a cancel that actually reaches the llama-server job (fixes the recorded Stop-doesn't-cancel + queue-races-repair bugs)
- Consecutive-action budget: pause after N auto tool calls for 'reviewed N — continue?', per-class toggles (auto reads, edits-in-workspace, pause on shell/MCP/net), post-edit settle delay
- Trajectory rewind & re-run: click a logged decision, edit the instruction/context there, re-run to compare counterfactual trajectories (reuse checkpoints + logRepair; teaches the KB the corrected path)
- (Lower priority) Agent Manager: parallel git-worktree agents with side-by-side diff/verify, respecting the LLM/SD VRAM mutex

## Cut list — deliberately off-mission (and why)

### Local-first LLM ecosystem tooling (llama.cpp / node-llama-cpp / Ollama / LM Studio / Jan) — making NexoraAI's local models more capable & reliable
**Key capabilities observed:**
- Speculative decoding is now mainstream in local runtimes. LM Studio 0.3.10/0.3.11 ships a draft-model UI (sidebar picker, per-main-model default, green-highlighted accepted tokens) and AUTO-INDEXES compatible main+draft pairs by tokenizer/vocab; measured 1.05x-2.43x speedups (2.43x on M3 Pro, 2.07x on RTX 3090 Ti). llama.cpp exposes it via --model-draft/--draft-max; best pairing is same-family, e.g. Llama-3.2-1B drafting Llama-3.1-8B (~1.83x) or a 0.5B draft for coding (up to ~2.5x). Jan v0.8.0 added the MTP (multi-token-prediction) variant using in-model prediction heads (DeepSeek-V3/R1) with no separate draft model.
- llama.cpp / node-llama-cpp support LAZY (triggered) grammars: a grammar stays dormant and only begins constraining tokens after a trigger word/regex/token is matched (waiting flag + trigger buffer). This is the mechanism that lets a reasoning model think freely, then emit a tool call whose ARGUMENTS are grammar-forced to the tool's JSON schema — reliability without breaking chain-of-thought. 'Strict' tool modes apply constrained decoding to arguments so a malformed-argument call becomes impossible.
- Grammar/JSON-schema constrained decoding is now near-free and the de-facto reliability layer for small models. XGrammar (default backend in many engines) does <40 microseconds/token, compiles a schema in 0.12-0.30s (vs Outlines' seconds), by splitting vocab into context-independent (precomputed masks) vs context-dependent tokens; XGrammar-2 (Jan 2025) adds a Cross-Grammar Cache reusing compiled substructures across related tool schemas — the key to cheap DYNAMIC tool sets. Benchmarks: with grammar active even small models produce perfect syntax; the only remaining errors are semantic. Ollama (Dec 2024) and LM Studio 0.3.0 both expose schema-constrained `format`/response_format built on this.
- llama.cpp tool calling matured via --jinja: a C++ Jinja2 engine auto-parses the model's chat template to detect its native tool-call format (JSON-native like Llama 3.1, or tag-based like Qwen/Hermes [TOOL_CALLS]/<tool_code>), with PEG-based tool-call parsing + JSON 'healing' for streaming. Native templates recognized for Qwen2.5, Mistral-Nemo, Llama-3.3, Functionary v3, Hermes 2/3, Granite; unknown templates fall back to a generic (token-heavier) format. WARNING documented: aggressive KV quant (e.g. -ctk q4_0) substantially degrades tool-calling quality.
- llama-server now has strong session/context persistence primitives. --slot-save-path enables /slots/{id}/save and /slots/{id}/restore REST endpoints to persist a conversation's KV cache to disk (per-conversation slot file). Since Oct 2025 --cache-ram is ON by default: idle slots are saved to host RAM and reused, giving up to ~93% TTFT reduction on cached prefixes; cache_prompt/similarity-based slot matching (default 0.5) reuses common prefixes so only the differing suffix is reprocessed. Community proxies persist KV to disk for ~7x faster restore of huge prompts.
- KV-cache quantization is the biggest memory knob: default FP16/BF16, but asymmetric quant is recommended — keep V-cache higher precision (drives output probabilities) and quantize K-cache more aggressively (relates tokens, degrades subtly). This is what lets long contexts fit on 4-8GB VRAM. Caveat: extreme KV quant hurts tool-calling; and --cache-ram must be tuned (or set 0) on 16GB/low-RAM machines to avoid starving the active cache or GGML_ASSERT crashes.
- Local RAG stack standardized in 2025-2026 around small on-device models: nomic-embed-text-v2 (137M, 274MB, 8K ctx, ~580 chunks/sec on CPU) as the fast default; BGE-M3 for multilingual + dense/sparse/ColBERT vectors; then a small cross-encoder reranker (bge-reranker-v2-m3 / BGE-reranker-large) adding ~3-5 MRR points at ~50ms/query. node-llama-cpp v3.19 natively supports BOTH embeddings AND reranking, so a local hybrid (vector + BM25) + rerank pipeline needs no cloud and no extra runtime. Contextual retrieval (prepend chunk context before embedding) + hybrid search fixes most retrieval failures.
- Reasoning/thinking control is now first-class locally: llama.cpp reasoning='auto'|on|off with a THINKING TOKEN BUDGET (-1 unrestricted, 0 off, N caps thinking tokens with an injected 'budget exhausted' message before </think>) and reasoning_format (deepseek → reasoning_content vs content). Ollama exposes think: true/false and low|medium|high effort (gpt-oss/Qwen3/DeepSeek-v3.1). Tool counts should stay under ~5-10 for 7B models (each tool def costs 50-150 tokens).
- node-llama-cpp 3.19 (the runtime NexoraAI already embeds) ships: generation-level JSON-schema enforcement, model-specific + generic function calling, embeddings, reranking, Jinja templates, smart context shift, token bias, LoRA, thought segmentation, prompt preloading — many of which NexoraAI does not yet exploit.

### Agentic Intent & Context-Engineering patterns (2025-2026): planning/decomposition, clarifying-question strategy, subagent orchestration, long-horizon memory, context engineering (compaction/retrieval/repo maps), self-verification, and reliable intent inference — mapped to concrete UX/architecture for NexoraAI's local-first + intent-based model.
**Key capabilities observed:**
- Uncertainty-aware clarification via a separate 'Intent Agent' (arxiv 2603.26233, 'Ask or Assume?'): a decoupled lightweight pass reads the turn's state history against 3 questions — intended goal? what is missing/ambiguous/assumption-heavy? what must be clarified before executing? — and, only when underspecification is detected, CONSTRAINS the main agent's next action to querying the user instead of coding. UA-Multi hit 69.4% resolve (vs 54.8% no-interaction baseline), well-calibrated (queried 344/500 tasks, ~3.06 q/task, 9.28% higher query rate on medium-difficulty vs easy). Single-agent 'reminder each turn' variant underperforms because it edits code before noticing gaps.
- Multi-interpretation single-response for ambiguity (arxiv 2511.10453, Nov 2025): instead of always asking or always guessing, the model emits up to 5 self-contained interpretation-answer pairs in one structured response (single pair when unambiguous). Trained with recall reward on ambiguous / precision reward on unambiguous. Crucially runs on Qwen3-4B-Instruct — a small local model — reaching 78-82% recall of valid interpretations. Directly applicable to on-device GGUF.
- Context engineering as the successor to prompt engineering (Anthropic 'Effective context engineering', AWS re:Invent AIM277 2025): (a) COMPACTION — summarize near limit, maximize recall of decisions/bugs/specifics first, then drop redundant tool output ('tool-result clearing' as lightest touch); (b) STRUCTURED NOTE-TAKING to files (NOTES.md/to-do) persisted OUTSIDE context; (c) JUST-IN-TIME retrieval — keep lightweight identifiers (paths/URLs/queries), load at runtime via tools rather than pre-stuffing; (d) 'context rot' begins well before the 200k ceiling, so keep high-signal context; (e) 'Goldilocks' prompting — start deliberately vague, add constraints only where testing reveals gaps.
- Context-Folding for long-horizon tasks (arxiv 2510.11967): hierarchical branch-and-fold — spawn an isolated sub-context for a subtask, complete it, then FOLD its full trace into a compact summary that feeds forward into the main context; selectively preserves decision-critical info and discards procedural detail. Distinct from flat compaction: it is recursive/hierarchical, so the main orchestration context stays small across many subtasks.
- Sub-agent orchestration, correctly scoped (Anthropic multi-agent research system; AOrchestra arxiv 2602.03786): orchestrator-worker where the lead explicitly passes CURATED context to each subagent; each subagent may burn tens of thousands of tokens exploring but returns only a 1-2k-token distilled summary — isolating detail from the lead's budget. Anthropic's 2025 refinement: subagents win for research-heavy UPFRONT work, not naive concurrent task-splitting (which underperformed); +90.2% over single-agent but ~15x tokens, so use judiciously.
- PageRank repo map / structural retrieval (Aider tree-sitter; LocAgent arxiv 2503.09089; Codebase-Memory arxiv 2603.27277): parse every file with tree-sitter to extract symbol defs + cross-file refs into a directed dependency graph, then PageRank-rank symbols with a 10x boost for identifiers mentioned in the conversation and 50x for in-chat files, emitting a compact ranked 'map' as just-in-time context. 2025 consensus: STRUCTURAL retrieval (relational queries over the symbol graph) is a complement to embedding retrieval (semantic similarity), not a replacement — and it needs no cloud embedding service.
- Interactive Plan-Mode as a durable, editable intent artifact (Cursor Plan Mode / 2.1 / 2.2, 2025; spec-driven development, Spec Kit Sept 2025, AGENTS.md standard): the plan is a MARKDOWN FILE the user edits inline (add/remove to-dos, adjust approach, add missed context) before any code is written; Cursor researches the codebase and asks clarifying questions via interactive UI during planning, can stream Mermaid diagrams into the plan, and can dispatch selected to-dos to new agents. Core principle: 'the spec is durable, the prompt is disposable' — intent written once, versioned, consumed by people and machines.
- Async priority steering + trajectory replay/counterfactual debugging (Claude Code h2A queue; AGDebugger, ACM CHI 2025; LangGraph time-travel): a real steering channel lets a typed correction fold into the running turn without stopping the tool in flight (queue holds several, editable via up-arrow) — distinct from a hard interrupt. AGDebugger enables editing a previously-sent message, resetting to an earlier point, and re-running to see the counterfactual trajectory — replaying WHY the agent chose a path, not just logging that it did.

**NexoraAI gaps:**
- No dedicated intent/underspecification pass. NexoraAI's flow is plan -> build -> verify/repro; it never decides 'this request is underspecified, ASK before writing' with calibration. It relies on keyword-demoted directives and one-shot generation, so an ambiguous 'make it look modern' or 'add auth' is executed on assumptions rather than triggering a smallest-necessary clarifying question.

### Local/On-Device Multimodal Frontier (image editing, speech-to-text, TTS, OCR, video) for a 4-8GB Electron+local-GGUF app
**Key capabilities observed:**
- Z-Image Turbo (Alibaba Tongyi, 6B, Apache-2.0, released 27 Nov 2025) runs text-to-image on a 4GB-VRAM GPU via stable-diffusion.cpp with Q4_K_M GGUF at 8-9 steps/CFG 1.0; leejet ships official leejet/Z-Image-Turbo-GGUF plus a dedicated 'How to Use Z-Image on 4GB VRAM' wiki. Quality reportedly matches/beats FLUX.1-dev and Qwen-Image on realism, at a fraction of FLUX's ~24GB. This is a drop-in upgrade over the SD1.5-class models NexoraAI currently defaults to.
- stable-diffusion.cpp (the SAME leejet binary NexoraAI already spawns as sd-server) supports img2img, inpainting, ControlNet, LoRA, PhotoMaker, ESRGAN upscaling, VAE tiling and Flash Attention for low VRAM — and, critically, its HTTP server exposes these over the API: init_image, mask_image, control_image, control_strength, strength params plus an OpenAI-compatible /v1/images/edits multipart endpoint and A1111-compatible layer. NexoraAI's localImageService.ts only ever calls /v1/images/generations (pure txt2img); the edit/img2img surface is running but unwired.
- Qwen-Image-Edit-2511 (instruction-based local image editing: 'make the jacket red', 'remove the person', multi-image composition) was updated on 26 Dec 2025 to work with stable-diffusion.cpp; unsloth ships Q4_K_M GGUF (~13GB total, runs on combined RAM+VRAM >13GB, CPU-capable). Also stable-diffusion.cpp now lists FLUX.1-Kontext-dev and Qwen-Image-2512 among supported edit/gen models.
- whisper.cpp (ggml-org) v1.8.x (2026) does real-time streaming dictation with built-in VAD (speech-segment detection cuts compute), quantized GGUF models (Q5_0, etc.), a --gpu-device flag, and an official whisper-server exposing an OpenAI-compatible /v1/audio/transcriptions HTTP endpoint (SSE streaming). tiny/base run faster-than-real-time on CPU-only (even Raspberry Pi); 0.5-2s latency behind live speech. This is architecturally identical to how NexoraAI already spawns llama-server and sd-server.
- Kokoro-82M (v1.0, StyleTTS2-based) is an 82M-parameter local TTS available in ONNX with fp32/fp16/q8/q4 quantizations, runs near-real-time on CPU/Apple Silicon, supports multiple languages (EN/FR/JA/KO/ZH) and per-voice embeddings you can blend. Note: Piper TTS (the old go-to) was ARCHIVED 6 Oct 2025 — Kokoro is now the leading small offline TTS.
- On-device OCR has bifurcated: Tesseract remains the only strong CPU-only option; PaddleOCR PP-OCRv4 (mobile/server, 80+ langs incl. CJK) beats Tesseract on layout/speed; and VLM-based OCR (Qwen2.5-VL-3B, Surya, olmOCR) gives structured Markdown/HTML/table output directly. NexoraAI ALREADY runs Qwen2.5-VL-3B via its vision sidecar (visionService.ts, port 8091) — so document OCR is a prompt away, no new model.
- Local text-to-video is now real but heavy: Wan2.2 5B fits ~8GB VRAM (~6 min/clip on RTX4090, much slower on weak GPUs), Wan2.2-14B GGUF runs from 6GB with offload, and LTX-2.3 runs locally via stable-diffusion.cpp. This sits at the very top of the 4-8GB range and is minutes-per-clip — a stretch/opt-in capability, not a core one, but the sd-server binary already technically supports Wan/LTX.
- Competitor baseline: Ollama shipped an official desktop app (Jul 2025) with image input for multimodal models; LM Studio and Jan focus on text/vision chat. None of Ollama/LM Studio/Jan ship integrated local image GENERATION, local voice dictation, local TTS, or OCR as first-class features — the common 2026 voice stack (Whisper+Piper+Ollama+Home Assistant) is a DIY glue job, not built in. ComfyUI/Krita AI cover local image editing but are not agentic IDEs. NexoraAI's integrated agent+local-image position is unusual; adding voice/edit/OCR would be genuinely differentiated.

**NexoraAI gaps:**
- localImageService.ts is text-to-image ONLY — it POSTs solely to /v1/images/generations. The sd-server it already spawns exposes init_image/mask_image/control_image/strength and /v1/images/edits, so NexoraAI cannot yet edit an image, do img2img from a reference, inpaint a masked region, or run ControlNet/LoRA despite the engine supporting all of it.
- No instruction-based image editing. Users can generate but not say 'make the sky orange' or 'remove the logo' on an existing asset — Qwen-Image-Edit-2511 now runs on the exact same binary but isn't in the catalog or wired.
- No speech-to-text / voice dictation anywhere (no MediaRecorder/getUserMedia/whisper spawn). Every prompt, chat message, and Universal-Agent intent must be typed. whisper.cpp is the natural sibling of the llama-server/sd-server pattern NexoraAI already uses.
- No text-to-speech. The agent, walkthroughs, and chat replies are text-only; no way to have results/plans read aloud (accessibility + hands-free). Kokoro-82M ONNX is tiny and CPU-friendly.
- No OCR / document intake, even though Qwen2.5-VL-3B is already downloaded and running as the vision sidecar. Users can't drop a screenshot of an error, a scanned spec, or a photo of a whiteboard and have text/structure extracted into a prompt or project.
- Default local image models are SD1.5-class (the routing regex and quality boosters target SD1.5). Z-Image Turbo (4GB-capable, far higher quality, official sd.cpp GGUFs) is not in imageCatalog.ts, so on-device quality lags the current frontier.
- Image directives are output-only. The model can emit [IMG] to generate, but there is no intent path to EDIT an existing project asset or reference photo (no [EDIT]/[MASK]-style intent), and no voice/OCR input directive to feed the intent layer new modalities.
